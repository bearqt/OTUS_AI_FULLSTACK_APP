Проект был сгенерирован одним промптом. Помимо него доработок больше не потребовалось (кроме фикса одной строчки Dockerfile). 
Использовал модель GPT-5.3-Codex (Reasoning - High) через расширение Codex.

Вот сам промпт:
# ROLE
Ты - Senior Fullstack Developer, эксперт по Python и React

# CONTEXT
MiniTrello - это веб-приложение, которое мы разрабатываем в рамках обучающего курса. Это приложение представляет собой аналог Trello (доска задач).

## FUNCTIONAL REQUIREMENTS
В приложении должно быть реализовано: 
1. Регистрация / авторизация (JWT)
2. Создание досок (boards), каждая доска — набор колонок
3. Колонки с настраиваемыми названиями (To Do / In Progress / Done и пользовательские)
4. Карточки: заголовок, описание (Markdown), приоритет (low/medium/high/urgent), дедлайн, метки
5. Drag-and-drop карточек между колонками и внутри колонки (сортировка)
6. Комментарии к карточкам с отметкой времени и автора
7. Фильтры: по приоритету, меткам, исполнителю, наличию дедлайна
8. Лог активности доски (кто, когда, какое действие)
9. Назначение карточки на пользователя (assignee)

## SEED DATA REQUIREMENTS
В приложении при старте должны быть следующие Seed-данные: 2 доски, 4 колонки, 20 карточек, 10 комментариев, 2 пользователя.

# TECHNOLOGY STACK

## Backend
1. Python (Django)
2. PostgreSQL
3. OpenAPI Swagger

## Frontend
1. React
2. Radix UI
3. Vite
4. eslint + prettier

## Deploy
1. Docker-compose (3 контейнера - Backend, Frontend, PostgreSQL DB)
2. GitHub Actions

# CRITICAL RULES AND RESTRICTIONS
1. При реализации приложения используй стэк технологий, указанный в секции "TECHNOLOGY STACK"
2. Всегда используй Context7 MCP когда требуется документация библиотеки или API, генерация кода или настройка конфигурации, даже если я явно этого не прошу.
3. Каждый метод API должен быть описан OpenAPI/Swagger документацией
4. Данные обязательно должны валидироваться и на Backend и на Frontend.

# TASK
Твоя задача - реализовать данное веб-приложение с нуля по требованиям, описанным выше.

## STEP-BY-STEP PROCESS:
1. Спроектируй схему БД для данного сервиса, в которой опиши все сущности, необходимые для полноценной работы приложения. Опиши ее в формате DBML и предоставь мне на согласование. Если я скажу, что эта схема корректна и больше не требует правок, то продолжай выполнение. Если я запрошу правки по этой схеме, реализуй их
2. Реализуй Backend часть на Python для данного приложения, согласно всем описанным требованиям в секции FUNCTIONAL REQUIREMENTS
3. Реализуй минимум 10 Unit тестов для Backend и убедись, что все они выполняются корректно.
4. Реализуй Frontend часть на React для данного приложения, согласно всем описанным требованиям в секции FUNCTIONAL REQUIREMENTS
5. Проверь сгенерированный проект на точное и полноценное соответствие всем описанным требованиям и отрефлексируй, насколько полно реализованы все запрашиваемые требования. Если какие-то требования не реализованы, реализуй их.
